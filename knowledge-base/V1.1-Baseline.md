## V1.1 Requirements

### Goal
Allow users to select from multiple online and local LLM models with easy switching.

### Scope
**Online Providers to Support**:
- ✅ ChatGPT (OpenAI) - already supported
- ✅ Claude (Anthropic) - already supported
- ➕ Gemini (Google)
- ➕ Perplexity

**Local Models**:
- Ollama integration with model discovery
- List downloaded models
- Download new models from Ollama library
- Select active model

### Key Behaviors
1. **Model Selection Scope**: Global default + per-conversation override
2. **Default State**: No model selected on first run - user must choose
3. **Model Switching**: Can change models mid-conversation in dashboard dropdown
4. **Persistence**: Settings and API keys survive restart
5. **Legacy**: Existing conversations default to GPT-4

### UI Flow
**Settings Page**:
- Section 1: Online Models (ChatGPT, Claude, Gemini, Perplexity)
  - Each provider has API key field
  - Model selection dropdown for each provider
- Section 2: Local Models (Ollama)
  - List of downloaded models with "Select" button
  - Search/download interface for new models
  - Current active model indicator

**Dashboard**:
- Dropdown showing all available models (online + local)
- Current selected model highlighted
- Can switch models mid-conversation

### First-Run Experience
On app startup with no model selected:
1. Show welcome screen
2. Guide user to select a model (online or local)
3. If online: prompt for API key
4. If local: show downloaded models or help download one
5. Once model selected, enable chat
